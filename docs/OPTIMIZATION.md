# Hermes 性能优化指南

## 问题分析

Hermes 响应慢的主要原因是：
1. **WSL2 启动延迟** - 每次调用需要 10-15 秒启动 WSL
2. **虚拟环境激活** - 每次都要重新激活 Python 虚拟环境
3. **进程创建开销** - 每次对话创建新进程

## 优化方案

### 方案 1: 预启动 WSL（推荐）

使用 `keep_wsl_alive.py` 保持 WSL 在后台运行：

```bash
# 方式1: 手动启动
python3 keep_wsl_alive.py

# 方式2: 使用快速启动脚本
start_fast.bat
```

**效果**: 响应时间从 15 秒降低到 3-5 秒

### 方案 2: 使用快速启动脚本

`start_fast.bat` 会自动：
1. 检查 WSL 状态
2. 如未运行，启动 WSL 保持服务
3. 等待 WSL 就绪
4. 启动 AI电脑管家

### 方案 3: 减少上下文长度

在 `hermes_bridge.py` 中：
- 限制对话历史到最近 3 条（原为 10 条）
- 减少处理时间

## 使用方法

### 快速启动（推荐）
1. 双击 `start_fast.bat`
2. 等待 WSL 启动（首次约 15 秒）
3. AI电脑管家自动启动
4. 使用 Hermes 时响应更快

### 手动保持 WSL 运行
1. 打开命令提示符
2. 运行: `python3 keep_wsl_alive.py`
3. 保持窗口运行
4. 启动 AI电脑管家
